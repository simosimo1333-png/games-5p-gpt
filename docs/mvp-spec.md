# 協力プレイMVP仕様

## 1. 目的

最初のオンライン版では、たくさんの機能を作ることより、「離れた2台のスマートフォンで同じゲームへ入り、協力し、最後まで一緒に遊べる」ことを証明する。

この文書では、最初に完成させる2人版を「2人縦切り」、その仕組みを最大6人へ広げた状態を「6人アルファ」と呼ぶ。

## 2. 決定事項

| 項目 | 2人縦切り | 6人アルファ |
| --- | --- | --- |
| 人数 | 2人固定 | 2〜6人 |
| 参加方法 | ルーム作成後、4〜6文字の合言葉を共有 | 同じ |
| 開始条件 | 2人が準備完了し、作成者が開始 | 2人以上が準備完了し、作成者が開始 |
| キャラクター | 全員が同じ基本能力 | 見分けやすい色を選択。能力差は設けない |
| 途中参加 | プレイ開始後は不可 | プレイ開始後は不可 |
| 制限時間 | 180秒 | 180秒を基準に人数別調整を計測 |
| クリア | 参加者全員がゴールへ到着 | 開始時の参加者全員がゴールへ到着 |
| 失敗 | 制限時間切れ | 同じ |
| 落下 | 直近チェックポイントへ1秒後に復帰 | 同じ |
| 切断 | 最大30秒間一時停止して再接続を待つ | 切断者のみ待機状態、残りは続行。30秒後にセッションから除外 |
| 再挑戦 | 参加者全員が同意すると同じステージを再開 | 過半数ではなく全員同意 |

## 3. 基本操作

- 左右移動
- ジャンプ

最初のMVPでは固有能力やアイテムボタンを追加しない。操作を増やさず、移動とジャンプだけで協力が生まれるステージを作る。

## 4. 最初のステージ

### テーマ

放課後の校舎から校門まで走る、1ステージ3分以内の横スクロールコース。

### 構成

1. **操作確認区間**: 安全な平地と低い足場で移動・ジャンプを確認する。
2. **合流区間**: 互いの位置と名前・色を確認する。
3. **協力ゲート**: 離れた2つの床スイッチを同時に1秒間踏むと、校門が永久に開く。
4. **復帰区間**: 落下しても直前のチェックポイントから戻れる足場を進む。
5. **ゴール区間**: 全員がゴール領域へ入るとクリアする。

## 5. 協力ギミック

最初のギミックは「2つの床スイッチ」とする。

- プレイヤー1人につき1つのスイッチを踏める。
- 2つが同時に押された状態を1秒維持すると、ゲートが開いて以後閉じない。
- スイッチの状態、1秒の計測、ゲート開放はサーバーが決定する。
- 片方の端末だけで開いて見える状態を許さない。
- 3〜6人版でも必要人数は2人とし、特定の役割や全員参加を必須にしない。

このギミックにより、特別な操作を増やさず「別々の場所へ移動してタイミングを合わせる」協力を検証できる。

## 6. ルームの流れ

1. 作成者が「ルームを作る」を選ぶ。
2. サーバーが推測しにくい合言葉を発行する。
3. 参加者が合言葉と表示名を入力する。
4. ロビーに参加者名、色、接続状態、準備状態を表示する。
5. 全員が準備完了すると、作成者が開始できる。
6. 3秒のカウントダウン後、同時にゲームを開始する。
7. クリアまたは時間切れ後、結果画面で再挑戦かロビー退出を選ぶ。

アカウント登録、フレンド、公開ルーム検索はMVPに含めない。

## 7. 切断と再接続

### 2人縦切り

- 片方が切断したらゲーム内時計と操作を一時停止する。
- 画面に「仲間の再接続を待っています」と残り時間を表示する。
- 30秒以内に同じ再接続トークンで戻れば、同じ位置と状態から再開する。
- 戻らなければセッションを終了し、接続中のプレイヤーをロビーへ戻す。

### 6人アルファ

- 1人の切断で全員を停止させない。
- 切断者を30秒間ゴースト表示し、安全な位置で操作不能にする。
- 再接続できなければ、そのプレイヤーをクリア必須人数から除外する。
- 残りが1人になった場合はセッションを終了してロビーへ戻す。

MVPではボット代行を実装しない。

## 8. 状態の正しさ

次の判断はサーバーを正とする。

- ルーム参加、準備、開始、終了
- プレイヤー位置と速度の妥当性
- 床スイッチとゲートの状態
- チェックポイントと落下復帰
- 制限時間、ゴール、クリア、失敗
- 切断猶予と再接続

クライアントは入力を送り、描画を滑らかにするための予測と補間を行う。

## 9. 2人縦切りの完了条件

- [ ] 異なる2台のスマートフォンが合言葉で同じルームへ参加できる
- [ ] 両者の準備後、同じカウントダウンで開始する
- [ ] 移動とジャンプが互いの画面へ同期される
- [ ] 2つの床スイッチを同時に踏み、両画面で同じゲートが開く
- [ ] 落下後、同じチェックポイントから復帰する
- [ ] 2人ともゴールした場合だけクリアになる
- [ ] 制限時間切れが両画面で同時に表示される
- [ ] 30秒以内の切断・再接続でゲームへ復帰できる
- [ ] 結果画面から同じメンバーで再挑戦できる
- [ ] iOS SafariとAndroid Chromeの組み合わせで完走できる

## 10. 6人アルファの完了条件

- [ ] 2〜6人が同じルームへ参加でき、7人目は理由つきで拒否される
- [ ] 6人の名前、色、接続状態を小さい画面でも確認できる
- [ ] 6人の移動が継続的に同期され、ゲーム進行が止まらない
- [ ] 任意の2人が協力ゲートを開ける
- [ ] 一時的に1人が切断しても残りのプレイが継続する
- [ ] 開始時の参加者全員、または猶予後に除外された人を除く全員の到着でクリアする
- [ ] 6台の実機で3回連続してセッションを完走できる

## 11. MVPに含めないもの

- 固有能力、職業、能力差
- 対戦、妨害専用行動、ランキング
- ランダムマッチング、公開ルーム一覧
- 途中参加、観戦、ボット代行
- アカウント、フレンド、チャット、ボイスチャット
- 課金、広告、アイテム収集
- 複数ステージと難易度選択

## 12. 今後決めること

MVPの実装を止めない項目として、別Issueで扱う。

- 6人版の固有能力と役割重複
- 対象年齢と表現ガイドライン
- 収益化と長期運営方針
- 本番バックエンドの地域と費用上限
- 180秒の制限時間とステージ難易度のプレイテスト調整
