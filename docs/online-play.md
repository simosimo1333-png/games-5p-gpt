# オンライン版の起動と配布

## ローカルで2台をつなぐ

最初のターミナルで画面を起動します。

```bash
npm run dev:client
```

2つ目のターミナルでゲームサーバーを起動します。

```bash
npm run dev:server
```

同じWi-Fi内のスマートフォンから、パソコンのIPアドレスとViteが表示したポートへアクセスします。ルーム作成後の「招待リンクをコピー」を使うと、合言葉入りのURLを共有できます。

## 本番配布

クライアントとWebSocketサーバーを別々に配布します。

- クライアント: 静的サイトとしてCloudflare Pages等へ配布
- サーバー: `apps/server/Dockerfile` を使い、常時起動とWebSocketに対応したサービスへ配布

クライアントURLへ `server` パラメーターを付けると接続先を指定できます。

```text
https://example.com/?server=wss%3A%2F%2Fgame-server.example.com
```

本番ではHTTPSのクライアントから接続するため、サーバー側もTLS対応の `wss://` URLが必要です。サーバーの外部ポートは配布先が設定する `PORT` 環境変数を使用します。

## 動作確認

1. 端末Aで表示名を入力し「ルームを作る」を押す。
2. 招待リンクを端末Bへ送る。
3. 端末Bで表示名を入力し「参加する」を押す。
4. 両方が「準備OK」を押し、作成者が「ゲーム開始」を押す。
5. 2人で床スイッチを踏み、ゲートを開け、全員で校門へ到着する。

一時的に通信が切れた場合、保存済みの再接続トークンを使って30秒間は同じプレイヤーとして復帰します。
