# ゲーム企画書

## 1. 概要

「放課後ダッシュ！」は、放課後の校舎や通学路を舞台に、最大6人で障害を乗り越えて制限時間内のゴールを目指すスマートフォン向けリアルタイム協力アクションゲームである。

## 2. プロダクトビジョン

短いプレイ時間の中で、「自分の成功」より「仲間を助けて全員で成功した瞬間」が記憶に残るゲームを作る。

## 3. 対象ユーザー

- 友人と3〜5分程度の短時間ゲームを遊びたい人
- 難しいコマンドより、移動・ジャンプ中心の直感操作を好む人
- 対戦の勝敗より、協力とハプニングを楽しみたい人

## 4. 対応環境

- 優先: iOS Safari、Android Chrome
- 補助: PCの主要ブラウザ（開発・観戦・検証用）
- 画面: 横向きを基本とし、端末回転時は案内を表示する
- 通信: 安定したWi-Fiまたはモバイル回線を想定する

## 5. コアループ

1. 合言葉または招待リンクでルームに集まる。
2. キャラクターまたは役割を選ぶ。
3. 3〜5分のステージへ挑戦する。
4. 移動、ジャンプ、能力を使い、仲間を助けながら進む。
5. 全員または規定人数がゴールして結果を確認する。
6. 同じメンバーで再挑戦し、タイムや救助回数を改善する。

## 6. 基本操作

- 左右移動
- ジャンプ
- キャラクター固有アクション1つ
- 状況に応じた救助・協力アクション

片手または両手で無理なく操作できるよう、同時入力数を抑える。

## 7. 協力の柱

### 役割の相互補完

例として、足場を作る、仲間を引き上げる、障害を止める、索敵するなど、単独でも役立ち、組み合わせると強くなる能力を用意する。特定キャラクター不在で進行不能にはしない。

### 救助と復帰

落下や失敗は即脱落ではなく、仲間の救助またはチェックポイント復帰につなげる。待ち時間を短くし、最後まで全員が参加できる状態を保つ。

### 全員での達成

最速の1人だけでなく、全員の到達、救助、協力アクションを結果に反映する。遅いプレイヤーを責める構造を避ける。

## 8. セッション仕様（初期案）

| 項目 | 初期方針 |
| --- | --- |
| 人数 | 2〜6人。開発初期は2人で検証 |
| 所要時間 | 1ステージ3〜5分 |
| 参加方法 | 合言葉、招待URL |
| 途中参加 | ロビーまで許可。プレイ中参加は後期検討 |
| 切断 | 短時間の再接続猶予後、ボットまたは安全停止を検討 |
| クリア | 規定時間内に全員がゴール。救済条件はプレイテストで調整 |
| 失敗 | 時間切れ。即再挑戦できる |

## 9. MVP

- 2人が同じルームへ入り、同じステージで互いを確認できる
- 移動とジャンプがサーバー経由で同期される
- 1種類の協力ギミックを2人で解ける
- 切断後に短時間で再接続できる
- ステージ開始、クリア、リトライが両者で一致する
- スマートフォン実機で安定して遊べる

MVP完成後に3〜6人、複数キャラクター、追加ステージへ広げる。

## 10. 非目標（初期段階）

- ランダム対戦やランキング競争
- 課金、広告、アイテム経済
- 大規模なアカウント・フレンド機能
- ボイスチャット
- ネイティブアプリ配信

## 11. 成功指標

- ルーム作成からゲーム開始までの成功率
- セッション完走率と再挑戦率
- 切断率、再接続成功率、クラッシュ率
- 1セッション中の協力・救助アクション回数
- モバイル端末におけるフレーム時間と入力遅延
- プレイテスト後の「仲間と協力できた」評価

## 12. 未決定事項

- 6人それぞれを固有役割にするか、役割重複を許可するか
- 1人切断時のAI代行または待機方式
- 全員ゴール必須か、救済ルールを設けるか
- 対象年齢、表現方針、収益化、運営期間
- バックエンドのホスティング先と地域

未決定事項は実装前にIssueで仮説、選択肢、決定期限を管理する。
