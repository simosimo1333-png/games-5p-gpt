# 対応端末と画面サイズ

## 方針

「放課後ダッシュ！」は、特定の端末名だけに合わせるのではなく、画面サイズに追従する設計にします。ゲーム画面は Phaser の `FIT` 拡大方式を使い、端末の表示領域からはみ出さないようにします。ノッチやホームインジケーター周辺には CSS の safe-area を使います。

## iPhone

2026年7月時点の最新上位モデルとして、次の画面サイズを自動テストの対象にしています。

| 端末 | Apple公表の画面解像度 | 自動テスト用の表示領域 |
| --- | --- | --- |
| iPhone 17 Pro | 1206 × 2622ピクセル | 402 × 874 CSSピクセル |
| iPhone 17 Pro Max | 1320 × 2868ピクセル | 440 × 956 CSSピクセル |

自動テスト用の値は、Apple公表の解像度を端末の画面倍率3で割った値です。実機固有のブラウザー表示領域を完全に再現するものではありませんが、WebKit、タッチ操作、縦横画面、safe-area、ゲーム画面の収まりを継続的に検査します。

## 自動テスト

GitHub Actionsでは、次を毎回確認します。

- Android Chrome相当でゲームが起動する
- iPhone 17 Pro相当のSafariでゲームが起動する
- iPhone 17 Pro Max相当のSafariでゲームが起動する
- 縦向きでは横向き案内を表示する
- 横向きでは案内を隠す
- ゲーム画面が端末の表示領域からはみ出さない

新しい主要端末が発売されたときは、Apple公表の仕様を確認してテスト設定を更新します。
