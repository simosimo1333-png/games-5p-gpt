# 現状分析と問題点

## 調査範囲

2026年7月12日時点の `main` ブランチを対象に、リポジトリ内の全ファイルとコミット履歴を確認した。

| ファイル | 状態 |
| --- | --- |
| `README.md` | リポジトリ名だけで、目的・実行方法・計画の説明がない |
| `index.html` | Phaser 3による1人用操作確認版。HTML、CSS、ロジック、ステージが1ファイルに同居 |
| `houkago-dash-starter.zip` | ソースと重複する可能性がある配布物。生成元、版、更新方法が不明 |

## 現行コードの動作

- 1280×720の固定論理解像度を `Phaser.Scale.FIT` で画面へ合わせる。
- 3,200px幅の単一ステージを横スクロールする。
- プレイヤーは矩形表示で、左右移動、ジャンプ、足場衝突、落下復帰がある。
- ゴール到達時に物理演算を停止し、タイムとリトライUIを表示する。
- キーボードと3つのタッチボタンをサポートする。
- Phaser 3.90.0をjsDelivrから実行時に取得する。

## 問題点

### P0: 企画と実装の不一致

- 企画は6人協力ゲームだが、現行コードはローカル1人用である。
- ルーム、参加・退出、同期、切断復帰、勝敗条件、協力ギミックが未定義・未実装である。
- 「6人」の役割、重複可否、途中参加、人数不足時の扱いが決まっていない。

### P0: 拡張しにくいコード構造

- HTML、CSS、ゲーム設定、入力、物理、UI、ステージデータが `index.html` に密結合している。
- `BootScene` が初期化、ゲーム進行、入力、UI、リトライをすべて担当している。
- 移動速度、重力、座標、色などがマジックナンバーとして散在する。
- ステージ追加やキャラクター追加のたびに中心ファイルが肥大化する。

### P0: オンラインゲームの技術基盤がない

- クライアントのみで、サーバー、プロトコル、権威モデルが存在しない。
- クライアント判定のまま通信を足すと、状態競合や不正操作へ対処できない。
- 遅延、パケット欠落、再接続、バージョン不一致を扱う設計がない。

### P1: 品質保証と開発環境がない

- パッケージ管理、型検査、Lint、Formatter、単体テスト、E2Eテストがない。
- CIがなく、変更後に起動できることを自動確認できない。
- エラー処理がなく、CDN読込失敗時はローディング画面のままになる。
- Phaserをバージョン固定している一方、依存の完全性や供給元障害への備えがない。

### P1: モバイル入力とアクセシビリティ

- `pointercancel` を扱わないため、OS割り込み時に移動状態が残る可能性がある。
- 同時タッチ、入力デッドゾーン、端末回転、セーフエリアの検証がない。
- 操作ボタンがPhaserオブジェクトで、読み上げやキーボードフォーカスに対応しない。
- `user-scalable=no` は利用者の拡大操作を禁止する。

### P1: ゲームプレイ上の不具合リスク

- 落下復帰地点は現在位置から220px戻すだけで、安全な足場を保証しない。
- 経過時間は `performance.now()` 基準で、ポーズやバックグラウンド遷移を考慮しない。
- リスタート時に入力状態が残るケースを網羅的に検証していない。
- 固定座標UIは、極端なアスペクト比やノッチ端末で操作性が落ちる可能性がある。

### P2: リポジトリ運用

- README、設計文書、ライセンス、貢献ガイドが不足している。
- ZIPをGit管理しており、生成物の正本が不明確で差分レビューができない。
- コミットメッセージが変更内容を説明していない。
- Issue、Pull Request、リリース、バージョニングの運用規約がない。

## 推奨するリファクタリング順序

1. 現行挙動を簡単なスモークテストと確認項目で固定する。
2. Vite、TypeScript、ESLint、Prettier、Vitest、Playwrightを導入する。
3. `index.html` から設定、シーン、入力、エンティティ、UI、ステージデータを分離する。
4. 1人用の機能同等性を確認してから、旧単一ファイル実装を退役させる。
5. 共有プロトコルと権威サーバーを追加し、まず2人の縦切りを成立させる。
6. 6人、再接続、マッチング、協力ギミックへ拡張する。

## 今回あえて変更しないもの

- `index.html` のゲームロジックと見た目
- Phaserのバージョンと読み込み方法
- `houkago-dash-starter.zip` の削除

これらは基準となる現行挙動や成果物の出所を確認した後、個別Issueで安全に変更する。
