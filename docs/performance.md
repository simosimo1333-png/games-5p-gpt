# 6人プレイの性能予算

## 目標

- サーバー: 20 tick/秒を維持し、1 tickの計算を25ミリ秒以内に収める
- snapshot: 1ルームあたり最大6人、20回/秒
- クライアント: 対象スマートフォンで操作中30 fps以上を維持する
- 通信: 1人あたり入力20回/秒以下、サーバーは30回/秒を超える入力を拒否する

## 自動測定

単体テストでは6人を参加させ、1,000 tickのシミュレーションを500ミリ秒以内で完了することを確認します。これはCIの性能回帰検出用で、実機の描画性能を置き換えるものではありません。

複数ルーム試験では、6人参加済みの50ルームを同時に100 tick更新し、合計1秒以内に収まることを確認します。

モバイルE2EではAndroid Chrome、iPhone 17 Pro相当、iPhone 17 Pro Max相当の3環境で、2端末の作成・参加・準備・開始を確認します。

## リリース前の実機確認

本番候補では6台を同じルームへ参加させ、3回連続でステージを完走します。フレーム落ち、入力遅延、切断、再接続成功率を記録し、この文書へ結果を追記します。
